# Running Ollama as a service

Exported agent graphs assume an Ollama daemon is reachable at
`http://localhost:11434` (or wherever `OLLAMA_HOST` points). For a one-shot
Python script you can just run `ollama serve` in another terminal — these
recipes are for leaving Ollama running in the background so the exported
script comes up cleanly every time.

## Windows — `nssm` wrapper

[NSSM](https://nssm.cc/) runs any `.exe` as a Windows service. Install
`nssm.exe` onto your `PATH`, then:

```cmd
:: 1. Register the service (run an elevated cmd.exe)
nssm install ollama "C:\Users\%USERNAME%\AppData\Local\Programs\Ollama\ollama.exe" serve

:: 2. Optional: set environment for GPU / host override
nssm set ollama AppEnvironmentExtra ^
    OLLAMA_HOST=127.0.0.1:11434 ^
    OLLAMA_NUM_PARALLEL=2

:: 3. Start it + set it to auto-start on boot
nssm start ollama
nssm set ollama Start SERVICE_AUTO_START
```

Stop / uninstall:

```cmd
nssm stop    ollama
nssm remove  ollama confirm
```

Logs: `nssm` writes stdout/stderr to whatever path you set via
`nssm set ollama AppStdout <path>`. By default they go nowhere.

## Linux — `systemd` unit

Drop this file at `/etc/systemd/system/ollama.service`:

```ini
[Unit]
Description=Ollama LLM daemon
After=network-online.target
Wants=network-online.target

[Service]
Type=exec
ExecStart=/usr/local/bin/ollama serve
User=%i
Restart=on-failure
RestartSec=3
Environment=OLLAMA_HOST=127.0.0.1:11434

[Install]
WantedBy=multi-user.target
```

Enable + start:

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now ollama
sudo journalctl -u ollama -f    # live logs
```

## Smoke test

Once the service is up, exported scripts should connect without any extra
setup. Verify with:

```bash
curl -s http://localhost:11434/api/tags | head -20
```

You should see a JSON list of installed models. Pull one if the list is
empty:

```bash
ollama pull qwen2.5:0.5b
```

## Exported-script env-var convention

Every script generated by the agents plugin reads `OLLAMA_HOST` at runtime
when the `Client(host=...)` call is made. To point the script at a remote
daemon without editing the source, set:

```bash
# bash
OLLAMA_HOST=http://10.0.0.5:11434 python my_agent.py

# Windows
set OLLAMA_HOST=http://10.0.0.5:11434 && python my_agent.py
```
